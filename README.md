## optimize3：检索算法重构与底层索引破局 (Retrieval & Algorithm)

本分支聚焦于解决 RAG 系统在海量长文本场景下的检索失效与成本高昂问题，落地了工业级混合检索架构。

*   **痛点 1：API 成本爆炸与 HNSW 索引拓扑坍缩**
    *   **问题**：传统的“父子块”双量入库会导致极高的 API Token 消耗。为降低成本，我切断了父块的 API 请求，对其注入 `[0.0] * 1024` 的统一假向量入库。但这引发了严重的“静默检索失效”：绝对一致的假向量导致 Milvus 底层的 HNSW（分层可导航小世界）图索引发生拓扑坍缩，形成了阻断遍历的“黑洞”。
    *   **方案**：下钻底层 C++ 引擎原理，摒弃绝对一致的假向量，在入库前向父块假向量注入 `random.uniform(-0.1, 0.1)` 的微小随机噪音。从数学拓扑层面强行打散节点分布，以极低的代码成本彻底摧毁了黑洞效应，恢复了图检索链路，使 API 成本骤降约 80%。

*   **痛点 2：“语义近视”与漏斗坍缩**
    *   **问题**：自动化评测显示纯向量模型的准确率仅有 75%左右，对具体数字、生僻专有名词极度不敏感。且早期的串行过滤（先向量后 BM25）导致关键子块如果在第一层未能召回，后续的精确过滤便沦为“无水之池”。
    *   **方案**：重构大厂标准的**并发双路召回**架构。并行执行向量语义召回（Top 60）与自研内存级全局 BM25 关键字捞取（Top 30），彻底打破漏斗坍缩。随后引入 RRF（倒数排序融合）算法进行高精度交叉打分，并接入 BGE-Reranker 进行终极语义精排，最终将系统的平均准确率从 75% 跃升至 97.5%。
